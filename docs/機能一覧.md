### F-01: 問い合わせフォーム

| 項目 | 内容 |
|------|------|
| Phase | 1 |
| 利用者 | お客様（外部公開） |
| 技術 | Power Apps（Canvas App） |
| kintone代替 | Form bridge |

#### 概要

お客様が施設利用の問い合わせを行うための入力フォーム。外部に公開し、URLをホームページに掲載する。送信内容はDataverseに自動登録される。

#### 入力項目

| 項目名 | データ型 | 必須 | 備考 |
|--------|---------|------|------|
| 会社名 | テキスト | ○ | |
| 担当者名 | テキスト | ○ | |
| メールアドレス | テキスト | ○ | メール形式バリデーション |
| 電話番号 | テキスト | ○ | |
| 希望施設 | 選択肢 | ○ | 笠間 / 木更津 |
| 希望日程（開始） | 日付 | ○ | |
| 希望日程（終了） | 日付 | ○ | 開始日以降のバリデーション |
| 参加人数 | 数値 | ○ | 1以上 |
| 希望サービス | 複数選択 | ○ | 会議室 / 食事 / 宿泊 |
| 備考 | テキスト（複数行） | − | 自由記述 |

#### 処理フロー

```
お客様がフォーム入力・送信
    ↓
入力バリデーション（クライアント側）
    ↓
Dataverse「問い合わせテーブル」にレコード作成（ステータス: 問い合わせ中）
    ↓
送信完了画面を表示
    ↓
F-04（問い合わせ受付通知）、F-05（受付確認メール）が自動起動
```

#### 画面仕様

- レスポンシブ対応（スマホ・タブレット・PC）
- 送信完了後、「受付番号」を画面に表示
- 空き状況カレンダー（F-03）からの遷移時は施設・日程を自動入力

### F-02: 会議室空き状況カレンダー（管理）

| 項目 | 内容 |
|------|------|
| Phase | 1 |
| 利用者 | スタッフ（社内） |
| 技術 | SharePointカレンダー |
| kintone代替 | − （新規） |

#### 概要

スタッフが**会議室**の空き状況をSharePointカレンダーに登録・管理するための画面。ブラウザから直接編集でき、現場スタッフが自分で運用できる。

> **💡 対象は会議室のみ**: 宿泊部屋の空き状況はaiPass（PMS）で管理しており、このカレンダーには含まない。宿泊部屋の空きは見積作成時にaiPassで都度確認する。

#### 空き状況データ（SharePointリスト）

| カラム名 | データ型 | 説明 |
|---------|---------|------|
| 施設 | 選択肢 | 笠間 / 木更津 |
| 日付 | 日付 | 対象日 |
| ステータス | 選択肢 | 空き / 残りわずか / 満室 |
| 残数 | 数値 | 残り利用可能枠数 |
| 備考 | テキスト | 午前のみ等の補足情報 |

#### 画面仕様

- SharePointカレンダービューで月表示
- 施設別（笠間 / 木更津）のフィルタリング
- ブラウザから直接編集（追加コスト¥0、E3/E5に含まれる）
- 色分け表示：空き（緑）/ 残りわずか（黄）/ 満室（赤）

#### Phase 3 での変更

aiPass連携（F-20）により、予約データから空き状況を自動計算。手動入力は不要になる。

### F-03: 会議室空き状況カレンダー（公開）

| 項目 | 内容 |
|------|------|
| Phase | 1（SharePointページ）→ 2（Power Pages） |
| 利用者 | お客様（外部公開） |
| 技術 | Phase 1: SharePointページ埋め込み / Phase 2: Power Pages（JS + Dataverse Web API） |
| kintone代替 | K viewer カレンダー |

#### 概要

お客様が**会議室の空き状況**を確認するためのカレンダー画面。Phase 1ではSharePointページに埋め込み、Phase 2ではPower Pagesで独立ページとして公開する。

> **💡 対象は会議室のみ**: 宿泊部屋の空き状況はこのカレンダーには含まない。

#### 画面仕様

- 月表示カレンダー
- 施設切替ドロップダウン（笠間 / 木更津）
- 日付ごとに空き状況アイコン表示：◎空き / △残少 / ✕満室
- 日付クリックで問い合わせフォーム（F-01）へ遷移（施設・日付を自動入力）
- レスポンシブ対応

#### Phase 2（Power Pages）での実装

- FullCalendar.js でカレンダーUI描画
- Dataverse Web API（`/_api/cr_availabilities`）で空き状況データを取得
- 匿名ユーザーでもアクセス可能（ログイン不要）
- Power Pages のキャッシュ（最大15分）を考慮し、Web API で直接取得

### F-06: 見積書PDF自動生成

| 項目 | 内容 |
|------|------|
| Phase | 1 |
| 利用者 | システム（自動処理） |
| 技術 | Power Automate + Wordテンプレート |
| kintone代替 | レポトン |

#### 概要

問い合わせ内容と料金マスタに基づき、Wordテンプレートに差し込んで見積書PDFを自動生成する。

#### 処理フロー

```
トリガー: スタッフが見積承認（F-08）を実行
    ↓
Dataverseから問い合わせレコード・料金マスタを取得
    ↓
Wordテンプレートにデータを差し込み
    ↓
Word → PDF変換（Power Automateの組み込みアクション）
    ↓
生成したPDFをDataverseの添付ファイル（Notes）に保存
    ↓
F-07（見積メール送信）を起動
```

#### 差し込みフィールド

| フィールド | データソース |
|-----------|-------------|
| 宛先（会社名・担当者名） | 問い合わせテーブル |
| 見積番号 | 自動採番 |
| 発行日 | 生成日 |
| 有効期限 | 発行日 + 30日（設定可） |
| 施設名 | 問い合わせテーブル |
| 利用日程 | 問い合わせテーブル |
| 参加人数 | 問い合わせテーブル |
| 項目明細（サービス・単価・数量・小計） | 料金マスタ × 問い合わせ条件 |
| 合計金額（税抜・税込） | 計算値 |

### F-07: 見積メール送信（日帰り/宿泊で分岐）

| 項目 | 内容 |
|------|------|
| Phase | 1 |
| 利用者 | お客様（受信側） |
| 技術 | Power Automate + Outlook |
| kintone代替 | K mailer |

#### 概要

生成した見積書PDFをお客様へ送信する。**日帰り利用と宿泊利用で処理が分岐**する。

#### 分岐ロジック

| 利用種別 | 処理 | 理由 |
|---------|------|------|
| **日帰り**（会議室のみ・食事のみ等） | 見積メールを**自動送信** 🔵 | 会議室の空きはSharePointカレンダーで確認済み |
| **宿泊あり** | Outlookに**メール下書きを作成**（PDF添付済み）→ 担当者が確認後に手動送信 🟠 | 宿泊部屋の空きはaiPass（PMS）でリアルタイム確認が必要 |

#### 判定条件

問い合わせテーブルの「希望サービス」に「宿泊」が含まれるかどうかで分岐する。

#### メール内容（共通）

| 項目 | 値 |
|------|-----|
| 送信元 | M365テナントのOutlookアドレス |
| 宛先 | お客様のメールアドレス |
| 件名 | 【施設名】お見積書をお送りします（受付番号: XXXX） |
| 添付 | 見積書PDF |
| 本文 | 見積内容の概要、有効期限、予約確定の案内 |

#### 日帰りの場合の後続処理

- ステータスを「見積送付」に自動更新
- スタッフへTeams通知（送付完了）

#### 宿泊の場合の後続処理

- Outlookの下書きフォルダにメールを作成（PDF添付済み）
- スタッフへTeams通知（「宿泊部屋の空き確認→メール送信してください」）
- 担当者がaiPassで宿泊部屋の空きを確認
- 担当者がOutlookの下書きを確認・送信
- 送信後、ステータスを「見積送付」に手動更新（またはPower Automateでメール送信を検知して自動更新）

### F-08: 見積承認フロー

| 項目 | 内容 |
|------|------|
| Phase | 1 |
| 利用者 | スタッフ |
| 技術 | Power Automate + Teams（承認コネクタ） |

#### 概要

スタッフがPower Apps管理画面で見積内容を確認し、承認操作を行う。承認後に見積書PDF生成（F-06）→ メール送信（F-07）が自動起動する。

#### 処理フロー

```
スタッフがPower Apps管理画面で見積内容を確認
    ↓
「承認」ボタン押下
    ↓
Power Automate 承認フローが起動
    ↓
承認者（上長等）にTeams通知（承認/却下の選択）
    ↓
承認 → F-06（見積書PDF生成）を起動
却下 → ステータスを「却下」に更新、担当者へ通知
```

### F-10: 条件変更・見積再送付

| 項目 | 内容 |
|------|------|
| Phase | 1 |
| 利用者 | スタッフ（操作）→ お客様（受信） |
| 技術 | Power Automate + Wordテンプレート + Outlook |

#### 概要

お客様からの条件変更依頼をスタッフがDataverseに反映すると、変更後の見積書PDFを自動再生成・メール送信する。

#### 処理フロー

```
Phase 1: お客様がメール・電話で変更連絡（手動）
Phase 2: お客様がマイページから変更リクエスト送信（F-16）
    ↓

    ↓
トリガー: Dataverse「見積・予約テーブル」のレコード更新
    ↓
変更後の見積書PDFを自動再生成（F-06と同じロジック）
    ↓
変更後の見積書をお客様へメール送信
    ↓
ステータスを「見積送付」に更新
```

#### Phase 2 での変更

お客様からの変更連絡が「メール・電話（手動）」から「マイページのセルフサービス（F-16）」に変わる。スタッフ側の操作は変更なし。

### F-20: aiPass連携（PMS同期）

| 項目 | 内容 |
|------|------|
| Phase | 3 |
| 利用者 | システム（自動処理） |
| 技術 | Rails（Render cron）+ Dataverse API |

#### 概要

aiPass（PMS: Property Management System）からDataverseへ予約データを定期同期する。空き状況の自動反映とaiPassへの手動入力の廃止を目的とする。

#### システム構成

```
aiPass（PMS）
    ↓ aiPass API（定期取得）
Rails（Render cron）
    ↓ データ整形・変換
Dataverse API
    ↓ 書き込み
Dataverse（空き状況テーブル等）
    ↓ 自動反映
Power Apps / Power Pages カレンダー
```

#### 同期内容

| 同期方向 | データ | 頻度 |
|---------|--------|------|
| aiPass → Dataverse | 予約データ（日程・人数・ステータス） | 定期（15分〜1時間ごと） |
| aiPass → Dataverse | 空き状況（自動計算） | 定期（予約データ同期と同時） |

#### Phase 3 で自動化される作業

| Phase 1-2（手動） | Phase 3（自動化） |
|---|---|
| 空き状況カレンダーへの手動入力（F-02） | aiPassデータから自動計算・反映 |
| aiPassへの予約手動入力 | Dataverse → aiPass の同期（将来検討） |
