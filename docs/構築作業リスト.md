# 新システム Phase 1 構築作業リスト

## 全体の進め方

```
STEP 1: 環境・データ設計
STEP 2: Dataverse構築
STEP 3: Power Apps構築
STEP 4: Power Automate構築
STEP 5: kintoneデータ移行
STEP 6: テスト・リリース
```

---

## STEP 1: 環境・データ設計

### 1-1. Power Platform環境の準備
- [ ] Power Platform管理センターで本番環境を作成
- [ ] 開発・テスト用の環境を別途作成（本番と分離）
- [ ] デジ推メンバーに管理者権限を付与
- [ ] 現場スタッフにPower Apps Premiumライセンスを付与

### 1-2. データ設計（Dataverse）
- [ ] kintoneの既存データ構造を確認・整理
- [ ] 以下のテーブル設計を決定
  - 問い合わせテーブル（顧客情報・希望条件・ステータス）
  - 見積・予約テーブル（金額・日程・サービス内容）
  - 空き状況テーブル（笠間・木更津・日付・空き数）
  - 顧客マスタテーブル
  - 料金マスタテーブル（平日・休日・前泊などの単価）
- [ ] テーブル間のリレーション設計
- [ ] ステータス遷移の定義（例：問い合わせ中→見積送付→予約確定→利用済→請求済）

### 1-3. 見積書・請求書テンプレート設計
- [ ] 現行レポトンの見積書レイアウトを確認
- [ ] Wordテンプレートの項目・デザインを決定
- [ ] 差し込みフィールドの定義（顧客名・日程・金額など）

---

## STEP 2: Dataverse構築

- [ ] 各テーブルをDataverseに作成
- [ ] 各テーブルのカラム（列）を設定
  - データ型の設定（テキスト・数値・日付・選択肢など）
  - 必須入力・バリデーションルールの設定
- [ ] テーブル間のリレーション（ルックアップ）を設定
- [ ] 選択肢フィールドの値を設定（施設名・サービス種別・ステータスなど）
- [ ] 料金マスタに初期データを登録
- [ ] セキュリティロールの設定（スタッフが参照・編集できる範囲を定義）

---

## STEP 3: Power Apps構築

### 3-1. 問い合わせフォーム（お客様向け・外部公開）
- [ ] Canvas Appで問い合わせフォームを作成
- [ ] 以下の入力項目を実装
  - 会社名・担当者名・連絡先
  - 希望施設（笠間 / 木更津）
  - 希望日程
  - 参加人数
  - 希望サービス（会議室・食事・宿泊）
- [ ] 入力バリデーションの実装
- [ ] 送信ボタン押下でDataverseへ登録
- [ ] 送信完了画面の実装

### 3-2. 空き状況カレンダー（お客様向け・外部公開）
- [ ] カレンダーUIの実装（月表示）
- [ ] 施設別（笠間・木更津）の切り替え
- [ ] 空き・残りわずか・満室の表示
- [ ] カレンダーから問い合わせフォームへの導線

### 3-3. 予約管理画面（スタッフ向け・社内）
- [ ] 問い合わせ一覧画面（ステータス別フィルター）
- [ ] 問い合わせ詳細・編集画面
- [ ] 見積内容の確認・承認画面
- [ ] 空き状況の手動入力画面
- [ ] 予約一覧・確定済み予約の確認画面

---

## STEP 4: Power Automate構築

### 4-1. 問い合わせ受付フロー
- [ ] フォーム送信をトリガーに自動起動
- [ ] 担当者へTeams通知を送信
- [ ] 自動受付確認メールをお客様へ送信（Outlook）

### 4-2. 見積書自動生成・送付フロー
- [ ] Wordテンプレートに問い合わせデータを差し込み
- [ ] PDF変換処理
- [ ] 見積メールをお客様へ自動送信（Outlook）
- [ ] スタッフへ送付完了をTeams通知

### 4-3. 見積承認フロー（スタッフ）
- [ ] スタッフがPower Apps管理画面で承認操作
- [ ] 承認後に見積送付フローを起動

### 4-4. 条件変更・見積再送付フロー
- [ ] Dataverse更新をトリガーに自動起動
- [ ] 変更後の見積PDFを再生成
- [ ] 変更内容をお客様へメール送信

### 4-5. 請求書自動生成・送付フロー
- [ ] 予約ステータスが「利用済」に変更されたタイミングで起動
- [ ] Wordテンプレートから請求書PDFを生成
- [ ] 請求書メールをお客様へ自動送信
- [ ] スタッフへ送付完了をTeams通知

---

## STEP 5: kintoneデータ移行

- [ ] kintoneから既存データをCSVエクスポート
- [ ] データのクレンジング・形式変換
  - 日付形式の統一
  - 顧客名・会社名の表記ゆれ修正
  - ステータス値の変換（kintone → Dataverseの選択肢に合わせる）
- [ ] DataverseへCSVインポート
- [ ] インポート結果の確認・修正

---

## STEP 6: テスト・リリース

### 6-1. 単体テスト
- [ ] 各Power Appsフォームの入力・送信テスト
- [ ] 各Power Automateフローの動作確認
- [ ] 見積PDF・請求書PDFの出力確認
- [ ] メール送信の確認（差し込み内容・レイアウト）

### 6-2. 結合テスト
- [ ] 問い合わせ→見積送付→予約確定の一連フローを通し確認
- [ ] 条件変更→見積再送付フローの確認
- [ ] 利用後→請求書送付フローの確認

### 6-3. ユーザーテスト
- [ ] 担当者に管理画面を実際に操作してもらう
- [ ] フィードバックを収集・修正

### 6-4. リリース準備
- [ ] 運用マニュアルの作成（担当者向け）
- [ ] 問い合わせフォームのURLをホームページに掲載
- [ ] kintoneからの切り替えタイミングを調整・決定
